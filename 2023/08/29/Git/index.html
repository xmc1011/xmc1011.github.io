Git

1、初始化配置

查看版本：git–version

省略（local）：本地配置，只对本地仓库有效

–global：全局配置，所有仓库生效

–system：系统配置，对所有用户生效

2、新建仓库

  • 本地：git init

.git是隐藏目录所以要用 ls -a

ls -altr

  • -a: 显示所有文件，包括以.开头的隐藏文件。
  • -l: 长格式输出，显示文件的详细信息，包括权限、所有者、大小等
  • -t: 按照修改时间排序，最新修改的文件排在前面。
  • -r: 倒序排序，与 -t 一起使用时，最新修改的文件排在最后面

指定目录：git init xxx 当前目录下会创建个xxx子目录

  • 远程服务器克隆：git clone

3、工作区域和文件状态

  • 工作区：.git所在的目录
  • 暂存区：**.git/index**用于临时存放即将提交的修改内容
  • 本地仓库：**.git/objects** Git存储代码和版本信息的主要位置

4、添加和提交文件

  • git init 创建仓库

  • git status 查看仓库的状态

  • git add 添加到暂存区

    • git add *****.txt——提交所有txt文件
    • git add **.**——提交所有目录下文件

  • git commit 提交 只会提交暂存区的文件，不会提交工作区的其他文件 需要使用 -m参数指定提交信息

  • git log 查看提交日志 –online简要提交信息

红色为未提交状态，绿色为提交了放入暂存区之后

5、git reset 三种模式

  • git reset –soft
  • git reset –hard
  • git reset –mixed （默认参数）

git ls-files查看暂存区的文件

6、git diff

  • 查看工作区、暂存区、本地仓库之间的差异
  • 查看不同版本之间的差异
  • 查看不同分支之间的差异

比较暂存区和 Git 工作区之间的差异

比较两个提交之间的差异：HEAD代表最新一次更新，HEAD^代表上一次版本

7、使用git rm删除文件

如果使用Linux下的rm进行删除，则需要在工作区先提交，在添加到暂存区，再提交

git rm

8、.gitignore

  • 系统或者软件自动生成的文件
  • 编译产生的中间文件或结果文件
  • 运行时生成日志文件、缓存文件、临时文件
  • 涉及身份、密码、口令、密钥等敏感信息文件

.gitignore文件的匹配规则

  • 空行或者以#开头的行会被Git忽略。一般空行用于可读性的分离，#一般用作注释

  • 标准的Blob模式匹配，例如：

    • 星号*通配任意个字符
    • 问号？匹配单个字符
    • 中括号[ ]表示匹配列表中的单个字符，比如：[abc]表示a/b/c

  • 两个星号**表示匹配任意的中间目录

  • 中括号可以使用短中线连接，比如

    [0-9]表示任意一位数字，[a-z]表示任意一位小写字母

  • 感叹号！表示取反

创建仓库，建立SSH

不要直接enter,根据提示是写入yes再回车

git push——从本地推送到GitHub

git pull——从GitHub推送到本地

10、关联本地仓库和远程仓库

origin：远程仓库的别名

git remote -v 查看当前仓库所对应的远程仓库的别名和地址

git push -u origin main:main 把本地的main分支推送给远程仓库main分支

git pull <远程仓库名> <远程分支名>：<本地分支名>

两个分支修改了同一文件，如果进行git merge会产生冲突，需要手动把需要留下的留下

13、回退和rebase

Merge

  • 优点：不会破坏原分支的提交历史，方便回溯和查看
  • 缺点：会产生额外的提交节点，分支图比较复杂

Rebase

  • 优点：不会新增额外的提交记录，形成线性历史，比较直观和干净
  • 缺点：会改变提交历史，改变了当前分支branch out的节点。避免在共享分支使用

分支命名：使用带有意义的描述性名称来命名分支

  • 版本发布分支/Tag示例：v1.0.0
  • 功能分支示例：feature-login-page
  • 修复分支示例：hotfix-#issueid-desc

分支管理

  • 定期合并已经成功验证的分支，及时删除已经合并的分支
  • 保持合适的分支数量
  • 为分支设置合适的管理权限
