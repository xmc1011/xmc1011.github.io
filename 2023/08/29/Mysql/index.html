二、标准语法

1、关键字

  • 关键字 SELECT...FROM...
  • 操作对象 ：表名、列名
  • 不区分大小写
  • 以分号结尾

语句类型

  1. DLL——数据定义语言
    1. CREATE创建数据库和表等对象
    2. DROP删除数据库和表等对象
    3. ALTER修改数据库和表等对象
  2. DML——数据操作语言
    1. SELECT 查询表中数据
    2. INSERT 像表中插入新数据
    3. UPDATE 更新表中数据
    4. DELETE 删除表中数据
  3. DCL——数据控制语言

    2. ROLLBACK 撤销SQL语句的执行结果

三、常用语法

1、单表查询

  • 单列查询 SELECT [列名] FROM [表名]；
  • 多列查询 SELECT [列名1], [列名2] FROM [表名]；
  • 所有列查询 SELECT * FROM [表名];

执行结果：数据表（二维表结构）查询结果和原表顺序一致

2、结果展示-SELECT

别名设置—AS

SELECT [列名1]，[列名2] AS [别名] FROM [表名] AS [别名]

场景：

  • 列名不符合规定，重新命名
  • 新生成的列

常数添加

  • 在结果中添加常数列：SELECT '常数' AS [别名] FROM [表名]；

    场景：添加特定值

四则运算

列与列之间、列与常数之间：SELECT [四则运算表达式] FROM [表名]；

场景：需要进行一定运算

字符拼接—CONCAT

将多个字段拼接成一个字段：SELECT CONCAT ([列名1], [列名2]) FROM [表名]

场景

  • 组合多列信息为一列
  • 对列的信息进行补充

指定字符拼接—CONCAT_WS

统一拼接符：SELECT CONCAT_WS('拼接符', [列名1],[列名2]) FROM [表名]

场景：

​ 多字段拼接，字段使用同样的拼接符

​ 会自动跳过NULL

内容去重—DISTINCT

取出不重复的列值：SELECT DINTINCT [列名] FROM [表名]

场景：

​ 列中有多个相同值，查看值的种类

结果排序—ORDER BY

  • 单列排序 : ...ORDER BY [列名];
  • 多列排序 :...ORDER BY [列名1],[列名2];
  • 列位置排序: ...ORDER BY [列位置1],[列位置2];

升序——ASC 降序——DESC

  • 是SELECT语句的最后一条子句
  • 排序列可以不显示
  • 排序只作用于前一列

CONVERT 拼音排序，将格式转为GBK

2、过滤语句——WHERE

过滤条件

  • 常规运算符 =/<> /!= /< /> /<= />=
  • 计算表达式 列与常数 列与列
    • FROM之后ORDER之前
    • 文本格式的数值，大小比较
    • NULL的处理

IFNULL 将空值设定为摸一个数在进行处理，空值设为15，则=15空值会显示出来

  • 数值过滤——BETWEEN…AND…

指定数值在某两个值之间：WHERE [列] BETWEEN [前端值] AND [后端值]

场景：

​ 简化取值范围的书写

​ 两端均包含

  • 空值过滤——IS NULL

判断列为空值：WHERE [列] IS NULL

场景：异常值处理

  • 非空值过滤——IS NOT NULL

判断列不为空值：WHERE [列] IS NOT NULL

场景：异常值处理

LIKE——模糊查询
  • 通配符—下划线_

​ 单个、任何字符：WHERE [列] LIKE '..._...';

​ 场景：知道字符数目

​ 知道字符位置

  • 通配符—百分号%

​ 任意数目（包括0个）、任何字符：WHERE [列] LIKE '...%...';

​ 场景：不知道字符数目

​ 知道字符位置

转义字符 \

  • “且”操作—AND

    两个或多个条件，同时满足：WHERE [条件1] AND[条件2] AND

    场景：不止一个过滤条件

    ​ 条件需要同时满足

  • “或操作”—OR

​ 两个或多个条件，至少满足一个：WHERE [条件1] OR [条件2] OR

​ 场景：不止一个过滤条件

​ 满足一个即可

  • 组合操作—AND…OR

​ 多个条件，不同要求：WHERE [条件1] AND [条件2] OR [条件3]

​ 场景：复杂情况

  • 取值限制—IN

    明确而不连续的取值：WHERE [字段] IN (值1，值2，...)；

​ 场景：过滤值明确

​ 不是连续范围

  • 否定条件—NOT

​ 否定过滤条件：WHERE NOT [条件1]；

​ 场景：否定一个或多个过滤条件

​ 不能单独使用

​ 类似补集的概念

只否定紧跟着的条件

1、数值型

  1. INTEGER
    1. 定义：整型
    2. 用法：INTEGER, INTEGER(10)
  2. DECIMAL
    1. 定义：定点型，存储固定位数的小数
    2. 用法：Decimal(2,2),DECIMAL(4,2)

    1. 定义：定长字符串

  1. DATE
    1. 定义：Y-m-d
    2. 用法：DATE
  2. TIME
    1. 定义：时间型 H:i:s
    2. 用法：TIME
  3. DATETIME
    1. 定义：Y-m-d H:i:s
    2. 用法：DATETIME

4、函数使用

函数表达式 = 函数名 + 参数

  1. 数值处理函数

    输出 （列） 决定函数功能 列名作用于该列的每一个单元格
    绝对值函数 ABS ABS(-3) = 3
    平方根函数 SQRT SQRT(4) = 2
    指数函数 EXP EXP(4) = e^4
    四舍五入函数 ROUND ROUND(1.234,2) = 1.23
    圆周率函数 PI PI () = pi

  2. 文本处理函数

    作用 名称 示例
    字符长度 CHAR_LENGTH CHAR_LENGTH(‘数据’) =2
    字节数 LENGTH LENGTH(‘数据’) = 6
    去除右边空值 RTRIM RTRIM(‘home’) = ‘home’
    去除左边空值 LTRIM LTRIM(‘home’) = ‘home’
    大写字母转换 UPPER UPPER(‘home’) = ‘HOME’
    小写字母转换 LOWER LOWER(‘HOME’) = ‘home’

  3. 日期处理函数

作用 名称 示例

获取具体时间段 HOUR/M/S HOUR(‘14:01:24’) = ‘14’

时间减少 DATE_SUB DATE_SUB(‘20190501’,INTERVAL 1 YEAR)

五、数据分析

1、聚合函数

函数表达式 = 聚合函数名 + 参数

输出（一行） 决定函数功能 列名汇总该列

不实际检索数据而是汇总处理数据

  1. 计数函数

    唯一可以用*的聚合函数

    • COUNT(*)/(1) 确定表中行的数目或符合特定条件的行的数目
    • COUNT(col) 只作用非NULL行 不可同时计数多列
    • COUNT(DISTINCT col) 对行去重计数

  2. 求和函数—SUM

    指定列的和：SUM(col)

    用法：区别于直接四则运算

    ​ 只对整数型作用

    ​ 可以同时操作多个列

    ​ 可以作用于计算表达式

  3. 均值函数—AVG

    指定列的平均值：AVG(col)

    ​ 用法：分母不对NULL计数

    ​ 可以同时操作多个列

  4. 最大值函数—MAX

    指定列的最大值：MAX(col)

    用法：适用于数值型、文本型、日期型

    ​ 要求该列可以排序

    ​ 忽略NULL

    ​ DISTINCT无意义

  5. 最小值函数—MIN

    指定列的最小值：MIN(col)

    用法：适用于数值型、文本型、日期型

    ​ 要求该列可以排序

    ​ 忽略NULL

    ​ DISTINCT无意义

2、分组数据

将数据分组，再在各组中进行聚合

  • 关键字 SELECT...FROM WHERE GROUP BY
  • 操作对象 必须子句 过滤条件 聚合键
  • 执行结果 多行数据表
  • SELECT组成
    • 聚合键、分组列：GROUP BY制定列
    • 聚合函数
    • 常数
  • 结果中一行数据即代表一个分组
  • select 展示列必须与分组对应，那么就只能是聚合键
  • NULL值单独成为一个分组
  • 聚合键尽量不要出现别名

3、分组过滤

将数据分组，各组中进行聚合，对聚合结果再次筛选，得到所需分组。一定在GROUP BY之后

  • 关键字 SELECT...FROM WHERE GROUP BY HAVING
  • 操作对象 必须子句 过滤条件 分组子句 分组筛选子句
  • 执行结果 多行数据表
    • WHERE指定‘行’的条件
    • HAVING指定“组”的条件
    • 分组之前排除的不参与分组
    • 聚合键写在WHERE中更适合

  • 书写顺序 ：

    SELECT...FROM... WHERE... GROUP BY... HAVING... ORDER BY...

    必须子句 过滤行 分组条件 筛选组 最后一条

    ​ 对应存在聚合函数 对应存在分组条件

  • 执行顺序:

    FROM... WHERE... GROUP BY... HAVING... SELECT... ORDER BY

尽量不要对GROUP BY使用别名，但是ORDER BY可以

4、表联结

关系型数据库将数据保存在不同的表中

原则：一类数据一个表，表之间相互联系

分解数据为多个表，更有效处理数据，但检索时，必须使用——联结

  • 联结：一种机制，关联多个表，返回一组输出，在一条SELECT语句中实现
  • 主键：唯一标识，可以是ID或任何其他唯一值

联结不是物理实体，在实际数据库表中不存在

SELECT 列名 指定检索列，可出自不同表
FROM 表1 ， 表2（联结的表） 从一个表拓展为跟多表
WHERE 表1.列a = 表2.列b
联结方式   联结字段		 过滤条件外，多一个联结条件
逐行匹配

1、内部联结

等值联结=

  • 联结前提：必须有共同列
  • 联结结果：所有表的所有列，共同重复的行
  • 创建联结规则：列出所有表、定义所有表关系
    • 所有表——INNER JOIN相当于，
    • 表关系——ON相当于WHERE

2、自联结

特殊内联结

  • 联结前提：联结表为同一个
  • 联结结果：返回所有数据
  • 创建联结规则：列出所有表、定义所有表关系

3、外部联结

  • 联结前提：必须有共同列
  • 联结结果：所有表的所有列，包含相关表中没有关联行的行
  • 创建联结规则：列出所有表、定义所有表关系
    • 左外部联结—包含左边表的所有行 LEFT OUTER JOIN
    • 右外部联结—包含右边表的所有行 RIGHT OUTER JOIN
    • 全外部联结—包含两边表的所有行 FULL OUTER JOIN
  1. 联结——先联结得到一个虚拟表 SELECT c.cust_id, c.cust_name, l.order_id FROM cust_info c,order_list l WHERE c.cust_id = l.cust_id
  2. 嵌套——对虚拟表二次查询 SELECT cust_id ,cust_name,order_id FORM( ) a;
  3. 聚合——在虚拟表基础上进行聚合操作 SELECT cust_id ,cust_name,COUNT(DISTINCT order_id) FROM( ) a GROUP BY cust_id;
  4. 简化——剥除虚拟表，留下作用语句

2、固化SQL语句—存储过程

存储过程：将多条语句保存为集合

具体操作 语句书写
新建一个表pet2，复制表pet内容 CREATE TABLE pet2 AS SELECT * FROM pet;
增加一个表字段AGE ALTER TABLE pet2 ADD age INT;
将表中字段AGE设置为一个常数t_num UPDATE pet2 SET age = t_num;
检索该表 SELECT * FROM pet2;
删除表 DROP TABLE pet2;

每次需要此操作时，都要逐条书写，并根据结果逐条修改

CREATE DEFINE = 'root'@ 'local host' PROCEDURE '<存储过程名>(输入输出类型 参数)'

BEGIN

  1. CREATE TABLE pet2 AS SELECT * FROM pet;
  2. ALTER TABLE pet2 ADD age INT;
  3. UPDATE pet2 SET age = t_num;
  4. SELECT * FROM pet2;
  5. DROP TABLE pet2;

END

输入输出类型 IN/OUT/INOUT

执行过程: CALL PROCEDURE <存储过程名>();

删除过程：DROP PROCEDURE <存储过程名>;

2、对字段的约束

  1. 主键—PRIMARY KEY

    <字段名> <数据类型> NOT NULL

    PRIMARY KEY

    ALTER TABLE <表名>

    ADD CONSTRAINT PRIMARY KEY (<列名>)

    作用—确保数据唯一性

    具体特征：

    • 每行都有（NOT NULL）
    • 行的唯一标识
    • 不能修改
    • 删除后不再分配

  2. 外键—FOREIGN KEY

    <字段名> <数据类型> NOT NULL

    REFERENCES KEY <关联表>(关联列)

    ALTER TABLE <表名>

    ADD CONSTRAINT FOREIGN KEY (<列名>)

    REFERENCE <关联表>(关联列)

    作用—确保引用完整性

    具体特征：

    • 该表与其他表的关联
    • 一定是其他表的主键

  3. 唯一约束—UNIQUE

    <字段名> <数据类型> UNIQUE

    ALTER TABLE <表名>

    ADD CONSTRAINT UNIQUE (<列名>)

    作用—确保行数据唯一性

    具体特征：

    • 可以有多个
    • 可以是NULL
    • 可以被更新
    • 可以重复使用
    • 不能定义外键

  4. 检查约束—CHECK

    <字段名> <数据类型> NOT NULL

    CHECK (<字段>(限定条件))

    举例：(sale_price >0)

    ALTER TABLE <表名>

    ADD CONSTRAINT CHECK (<字段>(限定条件))

    举例：(sex enum(‘M’,’F’))

    作用—确保该列数据满足指定条件

    具体特征：

    • 值的范围（>0）
    • 特定值（性别M/F）

  5. 默认约束—DEFAULT

​ <字段名> <数据类型> DEFAULT <默认值>

​ 作用—指定默认值

3、索引

建立索引 CREATE INDEX 索引名 ON 表名 (列名)

索引本质是一种排序

在某列上建立索引，即对该列进行”排序“

可以定义多列，实现组合排序

  1. 要求数据唯一性较好
  2. 索引提高查询，排序速度，适用于过滤或者排序较多的字段
  3. 数据修改（增删改）操作时，需要重新更新索引，效率降低
  4. 占用大量内存

  • 优势：提高检索效率，降低数据库IO成本，降低数据排序成本，降低CPU消耗
  • 劣势：索引实际是一张表，保存了主键和索引字段，在增删改的时候，MySQL不仅要更新数据，还要保存索引文件，降低表的更新速度

2、使用场景

  1. 主键自动建立唯一索引
  2. 频繁作为查询条件的字段（where后面的语句）
  3. 查询中与其他表关联的字段，外键关系建立索引
  4. 多字段查询下倾向创建组合索引
  5. 查询中排序的字段，排序字段若通过索引去访问大大提高排序速度
  6. 查询中统计或者分组字段
